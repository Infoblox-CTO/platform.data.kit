# Implementation Plan: Consolidate DataPackage Manifest

**Branch**: `004-consolidate-manifest` | **Date**: 2026-01-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-consolidate-manifest/spec.md`

## Summary

Merge `pipeline.yaml` into `dp.yaml` to provide a single manifest file per DataPackage. Add a new `spec.runtime` section for container configuration, implement automatic binding-to-environment-variable mapping, and support runtime overrides via `--set` and `-f` CLI flags. Remove all references to `pipeline.yaml` from examples and documentation.
