## 1. PackageRef (unified)

**Location**: `contracts/version.go`
**Replaces**: `contracts.ArtifactRef`, `controller.PackageRef`, `promotion.PackageRef`
**Purpose**: Single canonical reference to a specific version of a DK package artifact.

```go
// PackageRef is the canonical reference to a specific version of a
// DataKit package artifact. It contains enough information to pull
// exactly the right artifact from an OCI registry.
//
// This type unifies the former ArtifactRef (contracts), PackageRef
// (controller), and PackageRef (promotion) into a single definition.
type PackageRef struct {
    // Name is the package name (DNS-safe, lowercase).
    Name string `json:"name" yaml:"name"`

    // Namespace is the package namespace (optional, used by controller).
    Namespace string `json:"namespace,omitempty" yaml:"namespace,omitempty"`

    // Version is the exact resolved version (semver, no ranges).
    Version string `json:"version" yaml:"version"`

    // Registry is the OCI registry URL (e.g., "ghcr.io/infoblox-cto").
    Registry string `json:"registry,omitempty" yaml:"registry,omitempty"`

    // Digest is the OCI manifest digest (e.g., "sha256:abc123...").
    Digest string `json:"digest,omitempty" yaml:"digest,omitempty"`
}
```

### Validation Rules
- `Name`: Required, must match DNS-safe pattern `^[a-z][a-z0-9-]*$`
- `Version`: Required, must be valid semver (no ranges)
- `Registry`: Optional (defaults to configured registry)
- `Digest`: Required when used in lockfile entries; optional in constraint references
- `Namespace`: Optional (required only in controller context)

### Migration
- `ArtifactRef` becomes a type alias: `type ArtifactRef = PackageRef` with deprecation comment
- Controller's `PackageRef` import changes to `contracts.PackageRef`
- Promotion's `VersionFile.Spec.Package` type changes to `contracts.PackageRef`

## 2. LockFile

**Location**: `contracts/lockfile.go`
**Purpose**: Top-level lockfile structure serialized to `dk.lock`.

```go
// LockFile represents the dependency lock file for a DataKit package.
// It pins all dependencies to exact versions and OCI digests for
// reproducible builds. Written to dk.lock alongside dk.yaml.
type LockFile struct {
    // LockVersion is the schema version of the lockfile format.
    // Currently 1. Used for future format migration detection.
    LockVersion int `json:"lockVersion" yaml:"lockVersion"`

    // ManifestHash is the SHA-256 hash of the dk.yaml file content
    // at the time the lockfile was generated. Used to detect drift.
    ManifestHash string `json:"manifestHash" yaml:"manifestHash"`

    // GeneratedAt is the RFC3339 timestamp when the lockfile was generated.
    GeneratedAt string `json:"generatedAt" yaml:"generatedAt"`

    // Dependencies is an alphabetically-sorted list of locked dependencies.
    Dependencies []LockedDependency `json:"dependencies" yaml:"dependencies"`
}
```

### Validation Rules
- `LockVersion`: Required, must be `1` (for v1 schema)
- `ManifestHash`: Required, must start with `sha256:`
- `GeneratedAt`: Required, must be valid RFC3339
- `Dependencies`: May be empty (package with no dependencies)
- Entire file must be deterministic (sorted, fixed field order, trailing newline)

### File Characteristics
- **Filename**: `dk.lock`
- **Location**: Package root (same directory as `dk.yaml`)
- **Format**: YAML
- **Header comment**: `# Auto-generated by dk lock. DO NOT EDIT.`
- **Version control**: Should be committed (warned by `dk lint` if missing)

## 3. LockedDependency

**Location**: `contracts/lockfile.go`
**Purpose**: A single resolved dependency entry within the lockfile.

```go
// LockedDependency represents a single resolved dependency in the lockfile.
// It captures both the resolution result (exact version, digest) and the
// original constraint for validation and informational purposes.
type LockedDependency struct {
    // Name is the dependency name (matches the reference in dk.yaml).
    Name string `json:"name" yaml:"name"`

    // Kind is the dependency kind (Connector, Store, Asset).
    Kind Kind `json:"kind" yaml:"kind"`

    // Version is the exact resolved version (semver, never a range).
    Version string `json:"version" yaml:"version"`

    // Registry is the OCI registry URL where the artifact is stored.
    Registry string `json:"registry" yaml:"registry"`

    // Digest is the OCI manifest digest for content-addressable verification.
    Digest string `json:"digest" yaml:"digest"`

    // Constraint is the original version constraint from dk.yaml
    // (e.g., ">=1.0.0", "^2.1.0"). Stored for validation and display.
    Constraint string `json:"constraint" yaml:"constraint"`
}
```

### Validation Rules
- `Name`: Required, DNS-safe
- `Kind`: Required, must be valid `contracts.Kind`
- `Version`: Required, valid semver, no ranges
- `Digest`: Required, must match `sha256:[a-f0-9]{64}` pattern
- `Registry`: Required (explicit, not defaulted)
- `Constraint`: Required, must be a valid semver constraint expression

### Sorting
Dependencies are sorted alphabetically by `Name` for deterministic output.

## 4. CatalogEntry

**Location**: `contracts/catalog.go`
**Purpose**: Machine-readable metadata about a published package version. Stored in the OCI registry as manifest annotations and config blob fields.

```go
// CatalogEntry represents discoverable metadata for a published
// DataKit package version. It is stored alongside the OCI artifact
// in the registry (as annotations + config blob fields) and enables
// package discovery and filtering.
type CatalogEntry struct {
    // PackageRef contains the core package identification.
    PackageRef `json:",inline" yaml:",inline"`

    // Kind is the package kind (Connector, Store, Asset, Transform, AssetGroup).
    Kind Kind `json:"kind" yaml:"kind"`

    // Owner identifies the team or individual responsible for the package.
    Owner string `json:"owner,omitempty" yaml:"owner,omitempty"`

    // Description is a human-readable summary of the package.
    Description string `json:"description,omitempty" yaml:"description,omitempty"`

    // Tags are searchable labels for categorization.
    Tags []string `json:"tags,omitempty" yaml:"tags,omitempty"`

    // Classification is the data classification level.
    Classification string `json:"classification,omitempty" yaml:"classification,omitempty"`

    // CreatedAt is the RFC3339 timestamp when this version was published.
    CreatedAt string `json:"createdAt" yaml:"createdAt"`

    // Inputs are the assets consumed by this package (for Transforms).
    Inputs []AssetRef `json:"inputs,omitempty" yaml:"inputs,omitempty"`

    // Outputs are the assets produced by this package.
    Outputs []AssetRef `json:"outputs,omitempty" yaml:"outputs,omitempty"`

    // SchemaFingerprints maps asset names to their schema hashes.
    SchemaFingerprints map[string]string `json:"schemaFingerprints,omitempty" yaml:"schemaFingerprints,omitempty"`
}
```

### Storage Model (OCI Registry)

Catalog entry data is split across two OCI storage mechanisms:

**Manifest Annotations** (string key-value pairs, queryable without pulling blob):
| Annotation Key | Source Field |
|---------------|-------------|
| `dev.datakit.catalog.kind` | `Kind` |
| `dev.datakit.catalog.namespace` | `PackageRef.Namespace` |
| `dev.datakit.catalog.owner` | `Owner` |
| `dev.datakit.catalog.description` | `Description` |
| `dev.datakit.catalog.tags` | `Tags` (comma-separated) |
| `dev.datakit.catalog.classification` | `Classification` |
| `dev.datakit.catalog.created` | `CreatedAt` |

**ArtifactConfig** (JSON blob, requires pull to read):
| Config Field | Source Field |
|-------------|-------------|
| `inputs` | `Inputs` |
| `outputs` | `Outputs` |
| `schemaFingerprints` | `SchemaFingerprints` |

### Validation Rules
- `Kind`: Required, must be valid `contracts.Kind`
- `CreatedAt`: Required, valid RFC3339
- `PackageRef.Name`: Required
- `PackageRef.Version`: Required, valid semver
- `PackageRef.Digest`: Required
- `Owner`: Optional but recommended (warning if missing)
- `Classification`: Optional but recommended for Assets/Transforms (warning if missing)

## 5. Validation Error Codes

**Location**: `contracts/errors.go`
**Range**: E300–E319 (Lockfile), W300–W309 (Lockfile warnings)

| Code | Constant | Severity | Description |
|------|----------|----------|-------------|
| E300 | `ErrCodeLockfileStale` | Error | `dk.lock` manifest hash ≠ current `dk.yaml` hash |
| E301 | `ErrCodeLockfileOrphan` | Error | Lockfile has entry for undeclared dependency |
| E302 | `ErrCodeLockfileMissing` | Error | `dk.yaml` dependency has no lockfile entry |
| E303 | `ErrCodeLockfileConstraintViolation` | Error | Locked version doesn't satisfy dk.yaml constraint |
| E304 | `ErrCodeLockfileInvalidDigest` | Error | Digest format invalid (not `sha256:<hex64>`) |
| E305 | `ErrCodeLockfileInvalidVersion` | Error | Locked version not valid semver |
| W300 | `WarnCodeLockfileNotFound` | Warning | No `dk.lock` file found |

## 6. State Transitions

### Lockfile Lifecycle

```
                 dk init
                   │
                   ▼
           ┌──────────────┐
           │ No dk.lock   │
           │ (initial)    │
           └──────┬───────┘
                  │ dk lock
                  ▼
           ┌──────────────┐
           │ dk.lock      │◀──────────────┐
           │ (current)    │               │
           └──────┬───────┘               │
                  │                       │
          ┌───────┼───────┐               │
          │       │       │               │
     edit dk.yaml │  dk lock --update     │
          │       │       │               │
          ▼       │       └───────────────┘
   ┌──────────────┐
   │ dk.lock      │
   │ (stale)      │──dk lock──▶ (current)
   └──────────────┘
```

- **No dk.lock → Current**: `dk lock` resolves constraints and writes lockfile.
- **Current → Current**: `dk lock --update` re-resolves and overwrites.
- **Current → Stale**: User edits dk.yaml (manifest hash changes).
- **Stale → Current**: `dk lock` re-resolves and writes new lockfile.
- **Stale detection**: `dk lint` compares `manifestHash` in dk.lock with SHA-256 of dk.yaml.

### Catalog Entry Lifecycle

```
           dk publish
               │
               ▼
        ┌──────────────┐
        │ CatalogEntry  │
        │ (immutable    │
        │  per version) │
        └──────────────┘
```

- Catalog entries are **write-once per version** (immutability per existing ImmutabilityError).
- Re-publishing the same version fails with existing error handling.
- New version → new catalog entry with new digest.
